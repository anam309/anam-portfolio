Featured project

Retail Analytics Platform

Python · PostgreSQL · pandas · SQLAlchemy Personal build

A dimensional warehouse for retail transactions, with reporting on top

An end-to-end ETL pipeline that takes raw customer/product/sales extracts and turns them into a proper star schema: two SCD Type 2 dimensions with full change history, a fact table with pre-computed margin, and a data-quality framework that logs every rejected row instead of silently dropping it. Every run writes its own audit record (status, row counts, duration) and refreshes the materialized views behind a small business-reporting layer, so the pipeline's history is queryable and its output is report-ready, not just loaded.

SCD Type 2 modelingCustomer & product history preserved on every change, not overwritten.
Idempotent loadsSafe to rerun the same batch: hash comparison + dedup pre-filters, not just hope.
Data quality frameworkSix rejection categories, every failure logged with the full source row as JSON.
Business reportingRevenue trend, top products, customer LTV, and channel performance, refreshed on every run and exported to CSV/HTML.
Pipeline observabilityAn ETL audit log tracks status, row counts, and duration for every run.
Containerized & testedDocker Compose for local setup, 21 pytest tests, CI on GitHub Actions.
Debugged against realityFive real bugs found and fixed by actually running it against Postgres: documented, not hidden.
